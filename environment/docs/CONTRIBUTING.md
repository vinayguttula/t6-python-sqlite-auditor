# Contributing to the Release Readiness Auditor

We welcome contributions! Please follow these steps when contributing to the Auditor CLI:

1. **Environment Setup:** Ensure you have Python 3.12 installed. We recommend using a virtual environment (`python -m venv venv`).
2. **Code Style:** We follow the Black code style and use Ruff for linting. Please run `ruff check` before submitting a PR.
3. **Testing:** All new features must include pytest coverage. Run `pytest` locally.
4. **Pull Requests:** Open a PR against the `main` branch. Ensure your PR description clearly outlines the changes and links to any relevant Jira tickets.
